## 3. AuthForm — форма авторизации

### Что рассказывать
> «При запуске приложения открывается форма авторизации. Пользователь вводит логин и пароль. Введённые данные сверяются с таблицей Users. Если логин или пароль неверны, появляется сообщение об ошибке. Если пользователь заблокирован, вход запрещён.»

### Библиотеки и классы
- **Database.cs** — метод `AuthUser` выполняет SQL-запрос:
  ```sql
  SELECT Id, Login, IsAdmin, IsBlocked FROM Users WHERE Login = @login AND Password = @password
  ```
- **MySqlConnection, MySqlCommand** — открывают соединение и выполняют запрос.
- **MessageBox** — выводит сообщения об ошибках.

### Ролевая модель
- **IsAdmin = 1** — администратор, получает полный доступ.
- **IsAdmin = 0** — обычный пользователь, видит только таблицы и проходит капчу.
- **IsBlocked = 1** — пользователь заблокирован, вход невозможен.

### Пример данных
- Логин: `admin`, пароль: `admin` — администратор.
- Логин: `user`, пароль: `user` — обычный пользователь.

### Алгоритм
1. Читаем логин и пароль из текстовых полей.
2. Проверяем, что поля не пустые.
3. Вызываем `Database.AuthUser`.
4. Если пользователь не найден — сообщение «Неверный логин или пароль».
5. Если пользователь заблокирован — сообщение «Вы заблокированы».
6. Если обычный пользователь — открываем CaptchaForm.
7. Если администратор — открываем MainForm.

## 4. CaptchaForm — форма капчи

### Что рассказывать
> «Капча показывается обычным пользователям после успешной авторизации. Слева расположен квадрат 2×2 — целевые зоны. Справа находятся 4 клетки 2×2, в которых случайным образом появляются 4 кусочка изображения. Пользователь должен перетащить кусочки в левый квадрат так, чтобы получилось исходное изображение. Повороты не используются — только drag-and-drop.»

### Библиотеки и классы
- **PictureBox** — элементы для отображения зон и кусочков.
- **DragDropEffects** — эффект перетаскивания.
- **MouseEventArgs** — обработка нажатия мыши.
- **Random** — случайное распределение кусочков по правым клеткам.
- **CaptchaPiece** — класс, хранящий изображение и правильный индекс кусочка.

### Алгоритм
1. Загружаем 4 изображения из папки `Images`.
2. Каждому кусочку присваиваем `CorrectIndex` (0, 1, 2, 3).
3. Перемешиваем кусочки и помещаем в правые клетки (`spawn1`–`spawn4`).
4. При нажатии левой кнопкой мыши на клетку начинается drag-and-drop.
5. При_drop_ на левую зону происходит перенос или обмен кусочками.
6. При нажатии кнопки «Проверить» проверяется, что в каждой зоне находится кусочек с правильным индексом.
7. Если всё верно — открывается MainForm. Если нет — уменьшается счётчик попыток. После 3 неудач пользователь блокируется.

### Функции и что они делают
- `InitializeZones()` — добавляет левые зоны в список и делает их целями для drop.
- `InitializeSpawns()` — добавляет правые клетки в список и делает их источниками.
- `LoadCaptchaPieces()` — загружает изображения из папки `Images`.
- `InitializeCaptcha()` — случайно распределяет кусочки по правым клеткам.
- `Spawn_MouseDown()` — начинает перетаскивание из правой клетки.
- `Zone_DragEnter()` — разрешает drop на левую зону.
- `Zone_DragDrop()` — переносит или меняет местами кусочки.
- `btnCheck_Click()` — проверяет правильность сборки капчи.

## 5. MainForm — главное окно

### Что рассказывать
> «Главное окно открывается после прохождения капчи или сразу у администратора. В верхней части находится выпадающий список со всеми таблицами базы данных. При выборе таблицы её данные отображаются в DataGridView. Администратор может добавлять, удалять и сохранять строки. Обычный пользователь видит данные только для чтения. Также администратор видит кнопку «Валидация данных».»

### Библиотеки и классы
- **DataGridView** — таблица для отображения данных.
- **ComboBox** — список таблиц.
- **DataTable** — хранит загруженные данные таблицы.
- **MySqlDataAdapter** — загружает данные из MySQL в DataTable.
- **MySqlCommandBuilder** — автоматически генерирует команды UPDATE, INSERT, DELETE.
- **Database.cs** — методы `GetTables`, `GetTableData`, `SaveTableData`.

### Алгоритм
1. При загрузке формы вызывается `SetupUIByRole()` — настройка интерфейса по роли.
2. Загружается список таблиц через `Database.GetTables()`.
3. При выборе таблицы вызывается `LoadTableData()` — данные загружаются в DataGridView.
4. Кнопка «Обновить» перезагружает текущую таблицу.
5. Кнопки «Добавить», «Удалить», «Сохранить» доступны только администратору.
6. Кнопка «Валидация данных» открывает ValidationForm (только для администратора).

### Функции и что они делают
- `SetupUIByRole()` — настраивает доступность элементов по роли пользователя.
- `LoadTables()` — загружает список таблиц из базы данных.
- `LoadTableData()` — загружает данные выбранной таблицы.
- `FormatBooleanColumns()` — заменяет столбцы `IsAdmin` и `IsBlocked` на чекбоксы.
- `btnAdd_Click()` — добавляет новую пустую строку.
- `btnDelete_Click()` — удаляет выбранные строки.
- `btnSave_Click()` — сохраняет изменения в базе данных.
- `btnValidation_Click()` — открывает форму валидации.

### Ролевая модель
- **Администратор:**
  - Видит все таблицы, включая `Users`.
  - Может редактировать данные.
  - Видит кнопку «Валидация данных».
- **Обычный пользователь:**
  - Не видит таблицу `Users`.
  - Только просматривает данные.
  - Не видит кнопку валидации.

## 6. ValidationForm — форма валидации данных

### Что рассказывать
> «Форма валидации подключается к локальному эмулятору API TransferSimulator, работающему по адресу `http://localhost:4444/TransferSimulator/`. В текущей версии активна проверка ФИО. В коде уже подготовлены блоки для проверки СНИЛС, ИНН, email и удостоверения личности. Переключение между типами данных выполняется путём раскомментирования нужного блока.»

### Библиотеки и классы
- **HttpClient** — отправляет HTTP-запросы к API.
- **System.Net.Http.Json** — метод `ReadFromJsonAsync` для десериализации JSON.
- **System.Text.RegularExpressions** — регулярные выражения для проверки данных.
- **Message** — класс-модель ответа API с полем `value`.

### Алгоритм
1. При загрузке формы создаётся `HttpClient` с базовым адресом API.
2. При нажатии кнопки выполняется GET-запрос к нужному endpoint.
3. Полученное значение отображается в метке.
4. Вызывается соответствующий метод валидации.
5. Результат проверки отображается в другой метке.

### Функции валидации
- `GetValueAsync(string endpoint)` — универсальный метод получения данных из API.
- `ValidateFullName(string)` — проверка ФИО: только буквы, пробелы и дефисы.
- `ValidateSnils(string)` — проверка СНИЛС: 11 цифр.
- `ValidateInn(string)` — проверка ИНН: 10 или 12 цифр.
- `ValidateEmail(string)` — проверка формата email.
- `ValidateIdentityCard(string)` — проверка паспорта: 10 цифд.

### Поддерживаемые endpoint API
| Тип данных | Endpoint | Пример значения |
|---|---|---|
| ФИО | `fullName` | `Иванов Иван Иванович` |
| СНИЛС | `snils` | `12345678901` |
| ИНН | `inn` | `7707083893` |
| Email | `email` | `test@example.com` |
| Удостоверение личности | `identityCard` | `4515123456` |

## 9. SQL-запросы модуля 3

### Что рассказывать
> «В файле module3_query.sql находятся три варианта запроса для расчёта стоимости заказа. Первый вариант показывает детальный расчёт по каждой строке заказа. Второй — сводный расчёт по всему заказу. Третий — детальный расчёт для конкретного заказа. В расчёте учитывается количество продукции и стоимость материалов по спецификации.»

### Пример запроса
```sql
SELECT 
    o.OrderNumber AS 'Номер заказа',
    o.OrderDate AS 'Дата заказа',
    b.Name AS 'Покупатель',
    s.Name AS 'Исполнитель',
    o.Total AS 'Сумма заказа',
    COALESCE(SUM(m.Price * pm.Quantity * oi.Quantity), 0) AS 'Себестоимость материалов'
FROM Orders o
JOIN Companies b ON o.BuyerId = b.Id
JOIN Companies s ON o.ShopId = s.Id
JOIN OrderItems oi ON oi.OrderId = o.Id
LEFT JOIN ProductMaterials pm ON pm.ProductId = oi.ProductId
LEFT JOIN Materials m ON pm.MaterialId = m.Id
GROUP BY o.OrderNumber, o.OrderDate, b.Name, s.Name, o.Total;
```

### Почему себестоимость может быть 0
> «Если в заказе продукты, для которых нет спецификации в таблице ProductMaterials, то себестоимость материалов будет равна нулю. Например, в нашем заказе хлеб, а спецификация есть только для булочки с изюмом.»

## 12. Возможные вопросы преподавателя и ответы

**Вопрос:** Почему используется MySQL, а не MS SQL Server?  
**Ответ:** MySQL удобно использовать для локальной разработки, она была уже установлена, и задание не требовало конкретной СУБД.

**Вопрос:** Почему таблица называется `Companies`, а не `Customers` и `Shops`?  
**Ответ:** Для соблюдения 3NF и уменьшения дублирования. Одна организация может быть одновременно и покупателем, и продавцом. Флаги `IsSalesman` и `IsBuyer` позволяют определить её роль.

**Вопрос:** Как работает капча?  
**Ответ:** Кусочки изображения загружаются из папки `Images` и случайным образом распределяются по правым клеткам. Пользователь перетаскивает их в левый квадрат. Проверка сравнивает `CorrectIndex` каждого кусочка с номером зоны.

**Вопрос:** Как переключить валидацию на СНИЛС?  
**Ответ:** В `ValidationForm.cs` в методе `btnGetData_Click` закомментировать блок `fullName` и раскомментировать блок `snils`.

**Вопрос:** Как добавить новую таблицу в MainForm?  
**Ответ:** Достаточно создать таблицу в базе данных. `MainForm` автоматически подтянет её в выпадающий список, потому что используется `INFORMATION_SCHEMA.TABLES`.

**Вопрос:** Как восстановить базу данных на другом компьютере?  
**Ответ:** Создать базу `DemoExamDB` и выполнить `init_database.sql` или `database_dump.sql`.
